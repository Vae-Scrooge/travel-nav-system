# 项目迭代与优化方案

## 目标

按照“测试覆盖率提升、安全体系增强、架构升级与接口规范化”的优先级推进项目生产化改造。每一阶段均要求通过严格编译、自动化测试和文档更新后再进入下一阶段。

## 第一阶段：测试覆盖率提升工程

### 已完成

1. 将 `graph.c` 中的控制台 CRUD、手工建图、图打印和导游路线展示迁移到 `graph_console.c`。
2. 新增 `input.c`，将 `readString`、`readInt` 从 `global.c` 中拆出，避免输入层污染核心业务覆盖率。
3. 新增 `user_console.c`，将登录、注册和用户菜单交互从 `userManager.c` 中拆出。
4. 为图文件读写新增 `graphSaveToFiles`、`graphLoadFromFiles`，支持传入文件路径并返回执行结果。
5. 为用户管理新增 `saveUserToFile`、`validateUserInFile`，支持临时用户文件测试和旧明文密码迁移。
6. 扩展测试用例，覆盖文件不存在、参数文件格式错误、重复顶点、非法边、未知顶点、不可达路径、重复注册、登录失败和旧密码迁移。
7. 新增 `travels_console.c`，将交互式最短路径查询从 `travels.c` 迁出，核心算法模块只保留可测试逻辑。
8. 使用 `UNIT_TEST` 条件编译入口补充旧密码迁移异常路径测试，覆盖空参数、过长路径、源文件缺失、目标用户缺失和临时文件写入失败。

### 当前验证结果

| 指标 | 当前结果 |
|---|---:|
| 核心业务总覆盖率 | 92.57% |
| `global.c` | 100.00% |
| `graph.c` | 90.65% |
| `travels.c` | 93.66% |
| `userManager.c` | 93.98% |
| `password.c` | 90.12% |
| `stats.c` | 97.33% |

说明：整体核心覆盖率已超过 80%，且本轮重点文件 `graph.c`、`travels.c`、`userManager.c` 均已达到单模块 90% 以上目标。

### 后续测试优化任务

1. 已将 CI 覆盖率门禁从软失败调整为硬性门禁：整体不低于 80%，P0 核心模块不低于 90%。
2. 继续补充跨模块集成测试，覆盖真实菜单流程和数据文件组合异常。
3. 将测试运行输出中的历史中文编码乱码统一修复，提升报告可读性。
4. 评估是否引入 `gcovr` HTML 报告，便于定位剩余未覆盖分支。

## 第二阶段：安全体系增强

### 已完成

1. 新用户密码使用 PBKDF2-HMAC-SHA256 保存。
2. 登录时兼容旧明文密码。
3. 旧明文用户在验证成功后自动迁移为哈希格式。
4. 盐值生成改为优先使用系统级安全随机源：Windows 使用系统加密随机接口，其他平台使用 `/dev/urandom`。

### 待完成

1. 接入经过安全审计的 bcrypt、Argon2 或 PBKDF2 第三方库，替换项目内置实现。
2. 增加密码迁移日志，记录迁移成功、失败和异常原因。
3. 增加安全测试脚本，验证弱密码、重复盐值、错误哈希格式和迁移失败场景。
4. 建立用户数据备份和恢复流程，避免迁移失败导致数据丢失。

## 第三阶段：架构升级与接口规范化

### 当前判断

当前项目是本地 C 控制台程序，真实 RESTful API 并非立即必要。现阶段更适合先沉淀可复用业务函数，再根据是否需要多端访问、远程调用或前后端分离决定是否引入服务层。

### 如确认需要 RESTful 服务

1. 新增独立服务层，不直接复用控制台菜单。
2. 将现有 `graphAddSpot`、`graphRenameSpot`、`graphDeleteSpot`、`graphAddOrUpdateRoad`、`graphDeleteRoad`、`statsGetSnapshot` 封装为服务接口。
3. 统一请求、响应和错误码结构。
4. 为接口编写示例、错误响应说明和验收测试。
5. 控制台界面和服务层都调用同一批业务函数，避免两套逻辑分叉。

1. 严格编译通过：`-std=c99 -Wall -Wextra -Wpedantic -Werror`。
2. 自动化测试输出 `All tests passed.`。
3. 覆盖率报告更新到 `docs/TEST_REPORT.md`。
4. 静态分析执行完成，`docs/STATIC_ANALYSIS_REPORT.md` 和 `docs/cppcheck-target-report.txt` 已更新。
5. `README.md`、`CHANGELOG.md` 和相关设计文档同步更新。
