# 性能基准测试报告

## 测试目标

建立 Dijkstra 最短路径查询的基础性能基准，用于后续判断算法或数据结构修改是否造成明显性能回退。

## 测试程序

性能基准入口：

```text
tests/perf_benchmark.c
```

Makefile 目标：

```bash
make perf
```

该程序会构造不同规模和不同连通形态的图，并重复执行从第一个景点到最后一个景点的 Dijkstra 查询，输出 CSV 格式数据：

```text
scenario,nodes,edges,iterations,reachable,estimatedMemoryBytes,totalSeconds,averageMilliseconds
```

## 数据集设计

| 场景 | 景点数量 | 道路特点 |
|---|---:|---|
| 小规模 | 10 | 链式道路加少量跨越边 |
| 中规模 | 50 | 链式道路加中等数量跨越边 |
| 较大规模 | 100 | 接近当前 `MAXNUM` 上限，包含更多跨越边 |
| 随机稀疏图 | 80 | 固定随机种子生成稀疏无向边 |
| 密集图 | 60 | 接近完全有向图，用于观察高边数下表现 |
| 不可达图 | 80 | 两个互不连通分量，用于验证不可达路径性能 |

## 指标说明

| 指标 | 含义 |
|---|---|
| `nodes` | 景点数量 |
| `edges` | 有向道路记录数 |
| `iterations` | 重复查询次数 |
| `reachable` | 查询是否可达，1 表示可达，0 表示不可达 |
| `estimatedMemoryBytes` | 基于图结构和边节点数量估算的内存占用 |
| `totalSeconds` | 总 CPU 时间 |
| `averageMilliseconds` | 单次查询平均耗时 |

当前 C 标准库便携实现主要记录 CPU 时间和结构体估算内存。精确内存占用和系统级 CPU 使用率建议后续通过平台工具采集，例如 Windows Performance Monitor、Linux `/usr/bin/time -v` 或 CI 性能插件。

## 性能阈值建议

| 场景 | 建议阈值 |
|---|---:|
| 10 个景点 | 平均查询耗时 < 1 ms |
| 50 个景点 | 平均查询耗时 < 5 ms |
| 100 个景点 | 平均查询耗时 < 20 ms |
| 随机稀疏图 | 平均查询耗时 < 20 ms |
| 密集图 | 平均查询耗时 < 50 ms |
| 不可达图 | 平均查询耗时 < 20 ms |

如果后续任一场景平均耗时超过阈值，应检查是否出现重复矩阵转换、冗余文件 I/O、邻接表遍历退化或不必要的全图最短路径计算。

## 本地基准结果

本轮在 Windows + MinGW64 环境下执行 `tests/perf_benchmark.c`，输出如下：

| 场景 | 景点数 | 道路记录数 | 重复次数 | 可达 | 估算内存 | 总 CPU 时间 | 平均查询耗时 |
|---|---:|---:|---:|---:|---:|---:|---:|
| chain-small | 10 | 32 | 1000 | 1 | 3176 B | 0.000000 s | 0.000000 ms |
| chain-medium | 50 | 184 | 1000 | 1 | 6824 B | 0.002000 s | 0.002000 ms |
| chain-large | 100 | 376 | 1000 | 1 | 11432 B | 0.009000 s | 0.009000 ms |
| random-sparse | 80 | 320 | 1000 | 1 | 10088 B | 0.000000 s | 0.000000 ms |
| dense | 60 | 3540 | 500 | 1 | 87368 B | 0.011000 s | 0.022000 ms |
| unreachable | 80 | 156 | 1000 | 0 | 6152 B | 0.004000 s | 0.004000 ms |

说明：小规模场景可能低于 `clock()` 可观测精度，因此可能显示为 0。后续如需更精确结果，可提高迭代次数或使用平台高精度计时接口。
