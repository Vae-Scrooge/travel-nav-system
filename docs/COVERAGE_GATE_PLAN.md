## CI 门禁策略

第一阶段已经启用整体行覆盖率和分支覆盖率硬门禁：

```bash
gcovr --print-summary --fail-under-line 80 --fail-under-branch 80
```

该命令不得追加 `|| true`。当总体行覆盖率低于 80% 或总体分支覆盖率低于 80% 时，CI 必须失败并阻止合并。

第二阶段已经在 CI 中启用 P0 核心模块 90% 分模块门禁：

```bash
gcovr --filter '.*graph\.c' --fail-under-line 90
gcovr --filter '.*travels\.c' --fail-under-line 90
gcovr --filter '.*userManager\.c' --fail-under-line 90
```

第三阶段已经在 CI 中启用 P1 模块 90% 分模块门禁：

```bash
gcovr --filter '.*password\.c' --fail-under-line 90
gcovr --filter '.*stats\.c' --fail-under-line 90
```

## 分阶段提升计划

1. 第一阶段：整体覆盖率不低于 80%，当前已完成。
2. 第二阶段：P0 核心模块不低于 90%，当前已完成并接入 CI。
3. 第三阶段：将 P1 模块 `password.c`、`stats.c` 的 90% 门禁接入 CI，当前已完成。
4. 第四阶段：稳定保持分支覆盖率不低于 80%，连续多个迭代稳定后提升到 85%。
5. 第五阶段：为交互式流程增加端到端自动化测试，避免仅依赖单元测试覆盖率。

## 维护规则

1. 新增业务函数必须同步新增测试。
2. 修改密码、权限、路径算法和图文件读写时，必须补充异常路径测试。
3. 覆盖率下降超过 2 个百分点时，需要在变更说明中解释原因。
4. 禁止通过删除源码行、排除核心文件或软化 CI 门禁来规避覆盖率要求。
